      25             : /**
      26             :  *  Class to explore states reachable through threading.
      27             :  *  Runs an execution_statet that explores code containing threading functions,
      28             :  *  and when notified of context-switch generating operations, attempts to
      29             :  *  interleave threads in all possible ways.
      30             :  *
      31             :  *  To do this, the primary piece of information stored is a stack of
      32             :  *  execution_statets, with each ex_state representing a context switch point
      33             :  *  reached. A vector<bool> in each ex_state represents which context switches
      34             :  *  from this path have been explored already.
      35             :  *
      36             :  *  The algorithm is to run until the program completes, then feed the trace
      37             :  *  to the caller. From then on, when asked to generate a new trace we:
      38             :  *
      39             :  *    -# Remove the final ex_state from the stack
      40             :  *    -# Move to the new final ex_state on the stack
      41             :  *    -# Inspect whether we've explored all switches from this state.
      42             :  *      If yes, goto 1
      43             :  *    -# Pick a context switch to take from the current state, mark as explored
      44             :  *    -# Duplicate the final ex_state onto the end of the stack
      45             :  *    -# Move to the new end of stack; make the ex_state take the context
      46             :  *      switch we picked.
      47             :  *    -# Continue symbolic execution from here. Fin.
      48             :  *
      49             :  *  There are various scheduling possiblities. The default is depth-first
      50             :  *  search, where we just follow the algorithm above and return all the traces
      51             :  *  to the caller. The "schedule" way combines all paths into one trace, which
      52             :  *  is then solved once. Round-robin switches to the next thread in the set of
      53             :  *  threads when a context switch point occurs.
      54             :  *
      55             :  *  Some kind of scheduling interface/api would be good for the future.
      56             :  */
